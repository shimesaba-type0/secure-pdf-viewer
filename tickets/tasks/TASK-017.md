# TASK-017: 管理画面レスポンシブ対応改善

## 概要
管理画面のモバイル・タブレット対応の更なる改善とユーザビリティ向上

## 背景
現在の管理画面は基本的なレスポンシブ対応は実装済みですが、以下の課題があります：
- PDFファイル管理セクションでの操作性
- 小画面でのフォーム要素の使いやすさ
- ボタン配置の最適化

## 課題と改善点

### 1. PDFファイル管理セクションの改善
**課題:** 
- ファイルアクション（プレビュー、公開設定、削除）ボタンが小画面で縦に並ぶ際の配置
- ボタンサイズが不統一で操作しにくい

**改善案:**
```css
@media (max-width: 768px) {
    .file-actions {
        flex-wrap: wrap;
        gap: 0.5rem;
        width: 100%;
        margin-top: 0.5rem;
    }
    
    .file-actions .btn {
        flex: 1;
        min-width: 70px;
    }
}
```

### 2. フォームアクションボタンの改善
**課題:**
- 設定フォームのボタンが横並びで小画面では窮屈
- タップしにくいサイズ

**改善案:**
```css
@media (max-width: 768px) {
    .settings-form .form-actions {
        flex-direction: column;
        gap: 0.5rem;
    }
    
    .settings-form .form-actions .btn {
        width: 100%;
        min-height: 44px; /* タップしやすいサイズ */
    }
}
```

### 3. 管理セクションの余白調整
**課題:**
- モバイルでのパディングが大きすぎて画面を無駄遣い

**改善案:**
```css
@media (max-width: 768px) {
    .admin-section {
        margin-bottom: 1rem;
        padding: 1rem;
    }
}
```

## 実装対象ファイル
